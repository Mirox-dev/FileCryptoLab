# FileCryptoLab

**FileCryptoLab** — учебный консольный проект на C++ для демонстрации классических методов шифрования текста и работы с файлами.

Проект показывает базовые идеи криптографии, объектно-ориентированного программирования, разделения кода на `.h` и `.cpp` файлы, а также сборки проекта через CMake.

---

## Что делает проект

Программа работает с текстом из файла и демонстрирует процесс:

```text
исходный текст → шифрование → шифртекст → расшифровка → исходный текст
```

В текущей версии программа:

1. читает текст из файла `data/input.txt`;
2. создает объект выбранного шифра;
3. шифрует прочитанный текст;
4. расшифровывает полученный шифртекст;
5. выводит в консоль:
   - исходный текст;
   - зашифрованный текст;
   - расшифрованный текст.

В `main.cpp` сейчас для демонстрации используется **шифр Хилла**:

```cpp
HillCipher cipher(5, 8, 1, 3);
```

## Описание основных файлов

### `include/Cipher.h`

`Cipher` — базовый интерфейс для всех шифров.

```cpp
class Cipher {
public:
    virtual std::string encrypt(const std::string& text) const = 0;
    virtual std::string decrypt(const std::string& text) const = 0;

    virtual ~Cipher() = default;
};
```

Этот класс задает общее правило:

> Каждый шифр должен уметь шифровать и расшифровывать текст.

`Cipher` является абстрактным классом. Это значит, что объект `Cipher` напрямую создать нельзя. Он нужен как общий шаблон для конкретных шифров: `AffineCipher`, `HillCipher`, `GammaCipher`.

---

### `include/AffineCipher.h` и `src/AffineCipher.cpp`

`AffineCipher` реализует **аффинный шифр**.

Аффинный шифр работает по формуле:

```text
E(x) = (a * x + b) mod 26
```

Где:

- `x` — номер буквы в английском алфавите;
- `a` и `b` — параметры ключа;
- `26` — количество букв английского алфавита.

Расшифровка выполняется по формуле:

```text
D(y) = a^(-1) * (y - b) mod 26
```

Для корректной расшифровки число `a` должно быть взаимно простым с 26. Поэтому в конструкторе проверяется условие:

```cpp
if (MathUtils::gcd(a_, AlphabetSize) != 1) {
    throw std::invalid_argument(
        "Invalid affine key: parameter 'a' must be coprime with 26"
    );
}
```

Класс поддерживает:

- заглавные английские буквы;
- строчные английские буквы;
- сохранение пробелов, цифр и знаков препинания без изменений.

---

### `include/HillCipher.h` и `src/HillCipher.cpp`

`HillCipher` реализует **шифр Хилла** с матрицей 2×2.

Ключ задается четырьмя числами:

```cpp
HillCipher cipher(5, 8, 1, 3);
```

Эти числа образуют матрицу:

```text
5 8
1 3
```

Шифр Хилла работает с парами букв. Каждая буква переводится в число:

```text
A = 0
B = 1
C = 2
...
Z = 25
```

Затем пара чисел умножается на матрицу по модулю 26.

Особенности реализации:

- поддерживаются английские буквы;
- буквы приводятся к верхнему регистру;
- неанглийские символы сохраняются в тексте;
- если количество букв нечетное, при шифровании добавляется `X`;
- для расшифровки используется обратная матрица по модулю 26.

Перед использованием матрицы программа проверяет, существует ли обратная матрица:

```cpp
if (!hasInverseMatrix()) {
    throw std::invalid_argument("Hill cipher matrix must be invertible modulo 26");
}
```

Если обратной матрицы нет, такой ключ использовать нельзя.

---

### `include/GammaCipher.h` и `src/GammaCipher.cpp`

`GammaCipher` реализует **гаммирование**.

Гаммирование — это метод шифрования, при котором на открытый текст накладывается гамма, то есть последовательность символов. В данном проекте гамма формируется повторением ключа пользователя.

Например:

```text
text: Hello world
key:  secretsecre
```

Главная идея реализации:

```cpp
char encryptedSymbol = text[i] ^ key[i % key.length()];
```

Здесь:

- `text[i]` — текущий символ текста;
- `key[i % key.length()]` — текущий символ ключа;
- `^` — операция XOR.

Ключ повторяется по кругу за счет выражения:

```cpp
i % key.length()
```

Особенность операции XOR:

```text
text XOR key = encrypted
encrypted XOR key = text
```

Поэтому метод расшифровки вызывает тот же алгоритм:

```cpp
std::string GammaCipher::decrypt(const std::string& text) const {
    return encrypt(text);
}
```

Класс содержит поле:

```cpp
std::string key;
```

В нем хранится ключ, который передается в конструктор:

```cpp
GammaCipher::GammaCipher(const std::string& key) {
    this->key = key;
}
```

### `include/FileManager.h` и `src/FileManager.cpp`

`FileManager` отвечает за работу с файлами.

Он содержит два статических метода:

```cpp
static std::string readFile(const std::string& path);
static void writeFile(const std::string& path, const std::string& content);
```

`readFile` читает файл целиком и возвращает его содержимое в виде строки.

`writeFile` записывает строку в файл.

Файлы открываются в бинарном режиме:

```cpp
std::ifstream file(path, std::ios::binary);
std::ofstream file(path, std::ios::binary);
```

Бинарный режим полезен для шифрования, потому что шифртекст может содержать не только обычные читаемые символы.

---

### `include/MathUtils.h` и `src/MathUtils.cpp`

`MathUtils` содержит вспомогательные математические функции:

```cpp
int modulo(int value, int modulus);
int gcd(int left, int right);
int modularInverse(int value, int modulus);
```

Они используются в аффинном шифре и шифре Хилла.

#### `modulo`

Возвращает корректный остаток от деления, в том числе для отрицательных чисел.

#### `gcd`

Вычисляет наибольший общий делитель двух чисел.

#### `modularInverse`

Ищет обратное число по модулю.

Это нужно для:

- расшифровки в аффинном шифре;
- построения обратной матрицы в шифре Хилла.

### `main.cpp`

`main.cpp` — главный файл программы. Именно с функции `main()` начинается выполнение проекта.

Сейчас в нем демонстрируется работа шифра Хилла:

```cpp
HillCipher cipher(5, 8, 1, 3);

std::string text = FileManager::readFile("data/input.txt");
std::string encrypted = cipher.encrypt(text);
std::string decrypted = cipher.decrypt(encrypted);

std::cout << "Original:  " << text << '\n';
std::cout << "Encrypted: " << encrypted << '\n';
std::cout << "Decrypted: " << decrypted << '\n';
```

Порядок работы:

1. создается объект `HillCipher`;
2. из файла `data/input.txt` читается текст;
3. текст шифруется;
4. полученный шифртекст расшифровывается;
5. результат выводится в консоль.

## Как собрать проект

Для сборки используется CMake.

### 1. Создать папку сборки

```bash
mkdir build
cd build
```

### 2. Сконфигурировать проект

```bash
cmake ..
```

### 3. Собрать проект

```bash
cmake --build .
```

---

## Как запустить

Если сборка выполнялась через CMake из папки `build`, то запуск из папки `build`:

```bash
./FileCryptoLab
```

Но так как программа читает файл по пути:

```text
data/input.txt
```

удобнее запускать исполняемый файл из корня проекта:

```bash
./build/FileCryptoLab
```

## Теория: аффинный шифр

Аффинный шифр — это подстановочный шифр. Каждая буква заменяется другой буквой по формуле:

```text
E(x) = (a * x + b) mod 26
```

Для корректной работы параметр `a` должен быть взаимно прост с 26. Это нужно, чтобы существовало обратное число для расшифровки.

---

## Теория: шифр Хилла

Шифр Хилла — это блочный шифр, основанный на матрицах.

Текст разбивается на пары букв. Каждая буква переводится в число от 0 до 25. Затем пара чисел умножается на матрицу 2×2, после чего результат берется по модулю 26.

Для расшифровки нужна обратная матрица по модулю 26. Если обратной матрицы нет, такой ключ использовать нельзя.

---

## Теория: гаммирование

Гаммирование — это метод шифрования, при котором на открытый текст накладывается гамма.

В данном проекте гамма формируется повторением ключа пользователя. Каждый символ текста объединяется с символом ключа операцией XOR:

```text
encrypted[i] = text[i] XOR key[i % key.length()]
```

Для расшифровки используется тот же ключ и тот же алгоритм:

```text
decrypted[i] = encrypted[i] XOR key[i % key.length()]
```
